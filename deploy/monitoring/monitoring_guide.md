# Grafana Dashboard Creation Guide

This guide describes how to configure your Grafana dashboards to monitor API hits, system metrics, and container logs using your new Prometheus and Loki integrations.

---

## Step 1: Connect Data Sources in Grafana

Before building dashboards, make sure Grafana is connected to Prometheus and Loki:
1. Open Grafana at `http://localhost:3001` and log in (username: `admin`, password: `admin`).
2. Go to **Connections > Data Sources > Add data source**.
3. **For Prometheus**:
   * Select **Prometheus**.
   * Set the URL to: `http://prometheus:9090` (internal Docker network URL).
   * Scroll to the bottom and click **Save & test**.
4. **For Loki**:
   * Select **Loki**.
   * Set the URL to: `http://loki:3100` (internal Docker network URL).
   * Scroll to the bottom and click **Save & test**.

---

## Step 2: Create an "API Metrics" Dashboard

To create a new dashboard:
1. Click the **+** icon in the top right or go to **Dashboards > New Dashboard > Add a new panel**.
2. Select **Prometheus** as your query runner.

Here are the specific panels you should add:

### 1. Panel: "Total API Requests (Hits)"
* **Visualization**: **Stat** or **Time Series**
* **PromQL Query**:
  ```promql
  sum(http_requests_total)
  ```
* **Description**: Displays the total count of API requests received since the backend started.

### 2. Panel: "API Requests Rate (Hits/Sec)"
* **Visualization**: **Time Series**
* **PromQL Query**:
  ```promql
  sum(rate(http_requests_total[5m]))
  ```
* **Legend Format**: `Requests / Sec`
* **Description**: Displays the moving average rate of HTTP requests per second.

### 3. Panel: "API Hits by Endpoint/Route"
* **Visualization**: **Bar Chart** or **Time Series**
* **PromQL Query**:
  ```promql
  sum by (route) (rate(http_requests_total[5m]))
  ```
* **Legend Format**: `{{route}}`
* **Description**: Shows request rates broken down by individual routes (e.g., `/api/products`, `/api/health`).

### 4. Panel: "HTTP Status Codes (Success vs Errors)"
* **Visualization**: **Pie Chart** or **Time Series**
* **PromQL Query**:
  ```promql
  sum by (status_code) (rate(http_requests_total[5m]))
  ```
* **Legend Format**: `Status {{status_code}}`
* **Description**: Breaks down API responses by status code (e.g., `200` for success, `404` for not found, `500` for server errors). Helpful for detecting spikes in failures.

## Step 3: Create a "Node.js System Performance" Dashboard

Add a second dashboard or add a section to your main dashboard for Node.js backend performance metrics.

### 1. Panel: "Memory Usage (Heap)"
* **Visualization**: **Time Series**
* **PromQL Query**:
  ```promql
  nodejs_heap_size_used_bytes
  ```
* **Unit**: `Bytes (IEC)`
* **Description**: Tracks the actual memory consumed by the V8 engine heap inside the Node.js process. Great for detecting memory leaks.

### 2. Panel: "Event Loop Lag"
* **Visualization**: **Time Series**
* **PromQL Query**:
  ```promql
  nodejs_eventloop_lag_seconds
  ```
* **Unit**: `Seconds (s)`
* **Description**: Tracks the lag of the Node.js event loop. A high event loop lag indicates heavy CPU-bound tasks blocking the execution thread.

### 3. Panel: "Active TCP Sockets & Handles"
* **Visualization**: **Time Series**
* **PromQL Query**:
  ```promql
  nodejs_active_handles
  ```
* **Description**: Tracks the number of open sockets, files, and timer handles. High numbers can indicate unclosed database connections or TCP leaks.

## Step 4: Create a "Loki Logs Viewer" Panel

Using Loki, you can stream container stdout logs directly into your Grafana dashboard.

1. Add a **New Panel** on your dashboard.
2. Change the visualization type in the top right from *Time Series* to **Logs**.
3. Select **Loki** as the data source.

### 1. Panel: "Backend Live Logs"
* **LogQL Query**:
  ```logql
  {container="fullstack-backend"}
  ```
* **Description**: Displays the raw, streaming log output of the Node.js Express server container.

### 2. Panel: "Frontend Nginx Logs"
* **LogQL Query**:
  ```logql
  {container="fullstack-frontend"}
  ```
* **Description**: Displays the access and error logs generated by Nginx serving the Angular application.

### 3. Panel: "Log Error Frequency (Errors/Min)"
* **Visualization**: **Time Series**
* **LogQL Query**:
  ```logql
  sum(count_over_time({container="fullstack-backend"} |~ "error|fail|exception" [1m]))
  ```
* **Description**: Scans the backend logs for keywords like "error", "fail", or "exception" and graphs how many occur per minute.
